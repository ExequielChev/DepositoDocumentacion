Process Definition Document (PDD) — Descarga y Resumen de OPs

Fecha: 07/08/2025 · Área responsable: Servicios

1. Propósito

Definir el proceso actual manual y la solución automatizada mediante RPA para la descarga de OPs desde el sistema Major y su posterior análisis y exportación a Excel.

2. Alcance

  • Aplicable al proceso de generación de resumen de OPs a partir de datos extraídos del sistema Major.
  • Incluye desde la apertura del sistema hasta la generación de PDFs y Excel final.
  • Volumen de trabajo estimado: entre 10 y 30 OPs por día, dependiendo del momento del mes.

3. Actores y Roles

RolResponsabilidad
Usuario operadorEjecuta el bot, valida resultados y entrega Excel final.
Desarrollador RPARealiza el mantenimiento del script ante cambios en la interfaz o errores detectados.

4. Flujo actual AS-IS

  1. Ingresar al sistema Major → Contaduría → Pagos.
  2. Ingresar tipo y número de OP desde un Excel.
  3. Visualizar los datos de cada OP en pantalla.
  4. Transcribir los datos al Excel: PROVEEDOR, F.F, OC, BRUTO OP, NETO OP, DETALLE, FDO. AFECTADO, CF.
  5. Guardar los datos uno a uno.

5. Problemas frecuentes

  • Errores humanos al transcribir datos desde la pantalla al Excel.
  • Demoras operativas por volumen y necesidad de foco exclusivo del operador.

6. Requisitos funcionales esperados

  • Eliminar la necesidad de transcripción manual.
  • Automatizar la descarga de PDFs y exportación a Excel con los datos estructurados.
  • Permitir la ejecución desatendida del bot mientras el operador realiza otras tareas.
