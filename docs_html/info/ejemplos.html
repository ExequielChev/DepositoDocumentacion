Ejemplos de Documentación RPA

1. Process Assessment Document (PAD)

Ejemplo:

Proceso: Conciliación bancaria automática

Criterios de selección:

  • Volumen: 2,000 registros diarios
  • Reglas de negocio claras
  • Alto tiempo manual invertido (4 horas/día)
  • Bajo nivel de excepciones

Conclusión: Candidato viable para RPA, ROI en 4 meses.

2. Process Definition Document (PDD)

Ejemplo:

Nombre del Proceso: Registro de facturas proveedores

Aplicaciones utilizadas: Major, Excel

Pasos principales (AS-IS):

  1. Usuario descarga facturas desde correo.
  2. Valida datos en Excel.
  3. Ingresa manualmente en Major.
  4. Genera reporte de confirmación.

Excepciones: Factura con datos incompletos → enviar a supervisor.

3. Solution Design Document (SDD)

Ejemplo:

Arquitectura: Robot atendido en n8n + Orchestrator para colas.

Componentes:

  • Workflow de descarga de correos
  • Validador de datos en Excel
  • Bot de carga en Major
  • Generador de reportes PDF

Manejo de errores: Reintentos automáticos (3), logging en Orchestrator.

4. Testing Results

Ejemplo:

Prueba Resultado Esperado Resultado Obtenido Estado
Descarga de facturas Correo leído y guardado OK ✔
Carga en Major Factura registrada Error: campo vacío ✘

Conclusión: 8/10 pruebas superadas. Ajustar validación de campos vacíos.

5. UAT Sign-off

Ejemplo:

Proceso: Registro de facturas proveedores

Usuarios de negocio: Finanzas

Resultado: Proceso validado y aprobado para producción.

Firmas:

Usuario: _______________________
Gerente: _______________________
Fecha:   _______________________

6. Deployment Guide

Ejemplo:

Entorno: Producción - n8n Orchestrator

Pasos:

  1. Importar paquete .nupkg en Orchestrator.
  2. Asignar robot a cola "Facturas".
  3. Configurar credenciales en Orchestrator Assets.
  4. Activar trigger diario a las 8:00 AM.

7. Runbook / Manual Operativo

Ejemplo:

Robot: Bot_Registro_Facturas

Frecuencia: Diario, 8:00 AM

Acciones en caso de error:

  • Revisar logs en Orchestrator.
  • Si error de aplicación Major → Escalar a mesa de ayuda TI.
  • Si error de datos → Escalar a Finanzas.
