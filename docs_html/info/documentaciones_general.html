1. Identificación y Selección del Proceso

Objetivo: Detectar procesos candidatos para automatización que aporten valor.

Actividades:

  • Reuniones con stakeholders.
  • Evaluación de viabilidad técnica y económica.
  • Priorización de procesos.

Documentación generada: Process Assessment Document (PAD).

2. Mapeo y Análisis Detallado

Objetivo: Comprender a fondo el proceso actual (AS-IS).

Actividades:

  • Relevamiento con usuarios.
  • Identificación de aplicaciones, entradas y salidas.
  • Registro de excepciones y casos especiales.
  • Diagramación del flujo del proceso.

Documentación generada: Process Definition Document (PDD), diagramas de flujo.

3. Diseño de la Solución

Objetivo: Definir cómo se automatizará el proceso (TO-BE).

Actividades:

  • Definición de arquitectura RPA.
  • Selección de objetos reutilizables.
  • Definición de excepciones técnicas y de negocio.
  • Diagramas de alto nivel de la solución.

Documentación generada: Solution Design Document (SDD), High Level Design (HLD), Low Level Design (LLD).

4. Desarrollo (Build)

Objetivo: Construir el robot en la herramienta RPA.

Actividades:

  • Creación de objetos y workflows.
  • Implementación de validaciones y excepciones.
  • Configuración de logs y reportes.

Documentación generada: Código del proceso, Deployment Document, actualización del SDD.

5. Testing

Objetivo: Validar que el robot funciona correctamente.

Tipos de pruebas:

  • Unit Testing.
  • System Testing.
  • User Acceptance Testing (UAT).

Documentación generada: Testing Results, UAT Sign-off Document.

6. Deploy y Migración a Producción

Objetivo: Mover la automatización a producción.

Actividades:

  • Publicación del robot.
  • Configuración de colas y credenciales.
  • Piloto controlado en producción.

Documentación generada: Deployment Guide, Release Notes.

7. Soporte y Mantenimiento

Objetivo: Asegurar la continuidad del robot.

Actividades:

  • Monitoreo de ejecución.
  • Gestión de incidencias.
  • Ajustes ante cambios en sistemas o negocio.

Documentación generada: Runbook, Change Request Document (CRD), Incident Report.
